00032     //! \param[in] ctrinfo std::map of control file keywords and their info
00033     //! \details The ctrinfo map argument is optional. If not given, it is an
00034     //!    empty std::map constructed in-place and affects nothing. If given, it
00035     //!    contains the control file keywords, all of which are moved into the

00039     //!    example, because the command-line is parsed very early on during

00044     //!    it is still possible to request information on a control file
00045     //!    keyword, so it must be available. The input deck is where all
00046     //!    parsed information goes during control file parsing and is stored at
00047     //!    global scope, see, e.g., walker::g_inputdeck. This global-scope
00048     //!    (still namespace-scope), input deck object is thus created before
00049     //!    command-line parsing. The input deck object's constructor (working
00050     //!    only on type information, available at compile-time, of all the
00051     //!    control file keywords) creates a run-time map. This is a run-time
00052     //!    map, but available before main() starts, because it is const and it
00053     //!    is initialized as a global-scope map. This map is then passed in here
00054     //!    as ctrinfo, and its contents inserted into the CmdLine object, making
00055     //!    the control-file keywords and their info available during
00056     //!    command-line parsing. Since the input deck stack contains a copy of
00057     //!    the command-line stack, the command-line stack must be possible to be
00058     //!    instantiated without passing the ctrinfo map, otherwise it would be a
00059     //!    mutual dependency.
