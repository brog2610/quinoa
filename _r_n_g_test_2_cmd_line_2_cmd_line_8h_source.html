00054     //!   default constructor for the corresponding type. The ctrinfo map
00055     //!   argument is optional. If not given, it is an empty std::map
00056     //!   constructed in-place and affects nothing. If given, it contains the
00057     //!   control file keywords, all of which are moved into the relevant slot
00058     //!   (tag::ctrinfo). This allows constructing, e.g., a CmdLine object both
00059     //!   with and without this information in place, which are both used at
00060     //!   different stages of the execution. For example, because the
00061     //!   command-line is parsed very early on during runtime while the input
00062     //!   deck is only parsed much later, the control-file keywords and their
00063     //!   information (owned by and generated by the input deck and its
00064     //!   constructor) is not yet available when the CmdLine object is
00065     //!   constructed. However, during command-line parsing it is still possible
00066     //!   to request information on a control file keyword, so it must be
00067     //!   available. The input deck is where all parsed information goes during
00068     //!   control file parsing and is stored at global scope, see, e.g.,
00069     //!   walker::g_inputdeck. This global-scope (still namespace-scope), input
00070     //!   deck object is thus created before command-line parsing. The input
00071     //!   deck object's constructor (working only on type information, available
00072     //!   at compile-time, of all the control file keywords) creates a run-time
00073     //!   map. This is a run-time map, but available before main() starts,
00074     //!   because it is const and it is initialized as a global-scope map. This
00075     //!   map is then passed in here as ctrinfo, and its contents inserted into
00076     //!   the CmdLine object, making the control-file keywords and their info
00077     //!   available during command-line parsing. Since the input deck stack
00078     //!   contains a copy of the command-line stack, the command-line stack must
00079     //!   be possible to be instantiated without passing the ctrinfo map,
00080     //!   otherwise it would be a mutual dependency.
00081     //! \author J. Bakosi
00082     CmdLine( tk::ctr::HelpFactory ctrinfo = tk::ctr::HelpFactory() ) {
00083       set< tag::verbose >( false ); // Use quiet output by default
00084       // Initialize help: fill from own keywords + add map passed in
00085       boost::mpl::for_each< keywords >( tk::ctr::Info( get< tag::cmdinfo >() ) );
00086       get< tag::ctrinfo >() = std::move( ctrinfo );
00087     }
