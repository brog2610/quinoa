      52                 :            : /*! The Epetra_SerialDenseVector class enables the construction and use of real-valued,
      53                 :            :     double-precision dense vectors.  It is built on the BLAS and LAPACK and derives from the Epetra_SerialDenseMatrix class.
      54                 :            : 
      55                 :            : The Epetra_SerialDenseVector class is intended to provide convenient vector notation but derives all signficant
      56                 :            : functionality from Epetra_SerialDenseMatrix.
      57                 :            : 
      58                 :            : <b>Constructing Epetra_SerialDenseVector Objects</b>
      59                 :            : 
      60                 :            : There are four Epetra_SerialDenseVector constructors.  The first constructs a zero-length object which should be made
      61                 :            : to appropriate length using the Size() or Resize() functions and then filled with the [] or () operators.
      62                 :            : The second constructs an object sized to the dimension specified, which should be filled with the [] or () operators.
      63                 :            : The third is a constructor that accepts user
      64                 :            : data as a 1D array, and the fourth is a copy constructor. The third constructor has
      65                 :            : two data access modes (specified by the Epetra_DataAccess argument):
      66                 :            : <ol>
      67                 :            :   <li> Copy mode - Allocates memory and makes a copy of the user-provided data. In this case, the
      68                 :            :        user data is not needed after construction.
      69                 :            :   <li> View mode - Creates a "view" of the user data. In this case, the
      70                 :            :        user data is required to remain intact for the life of the object.
      71                 :            : </ol>
      72                 :            : 
      73                 :            : \warning View mode is \e extremely dangerous from a data hiding perspective.
      74                 :            : Therefore, we strongly encourage users to develop code using Copy mode first and
      75                 :            : only use the View mode in a secondary optimization phase.
      76                 :            : 
      77                 :            : <b>Extracting Data from Epetra_SerialDenseVector Objects</b>
      78                 :            : 
      79                 :            : Once a Epetra_SerialDenseVector is constructed, it is possible to view the data via access functions.
      80                 :            : 
      81                 :            : \warning Use of these access functions cam be \e extremely dangerous from a data hiding perspective.
      82                 :            : 
      83                 :            : 
      84                 :            : The final useful function is Flops().  Each Epetra_SerialDenseVector object keep track of the number
      85                 :            : of \e serial floating point operations performed using the specified object as the \e this argument
      86                 :            : to the function.  The Flops() function returns this number as a double precision number.  Using this
      87                 :            : information, in conjunction with the Epetra_Time class, one can get accurate parallel performance
      88                 :            : numbers.
