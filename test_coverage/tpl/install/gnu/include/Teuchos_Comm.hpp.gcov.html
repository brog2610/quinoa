     152                 :            : /// \class Comm
     153                 :            : /// \brief Abstract interface for distributed-memory communication.
     154                 :            : /// \tparam Ordinal Type of indices used for communication.
     155                 :            : ///
     156                 :            : /// \section Teuchos_Comm_What What is Comm?
     157                 :            : ///
     158                 :            : /// This class is Teuchos' interface to distributed-memory
     159                 :            : /// communication between one or more parallel processes.  It presents
     160                 :            : /// an interface very much like that of MPI (the Message Passing
     161                 :            : /// Interface).  Teuchos provides two implementations of Comm:
     162                 :            : /// - An MPI (Message Passing Interface) implementation, MpiComm
     163                 :            : /// - A "serial" implementation, SerialComm, that only has one process
     164                 :            : ///
     165                 :            : /// Comm is an abstract interface.  You cannot create a Comm directly.
     166                 :            : /// You have to create one of the subclasses.  The normal way to
     167                 :            : /// handle a Comm is to pass it around using RCP (a reference-counted
     168                 :            : /// "smart" pointer).  For example:
     169                 :            : /// \code
     170                 :            : /// // Make a Comm.  This one happens to wrap MPI_COMM_WORLD.
     171                 :            : /// RCP<const Comm<int> > comm = rcp (new MpiComm (MPI_COMM_WORLD));
     172                 :            : /// // Equivalent of MPI_Comm_rank
     173                 :            : /// const int myRank = comm->getRank ();
     174                 :            : /// // Equivalent of MPI_Comm_size
     175                 :            : /// const int numProcs = comm->getSize ();
     176                 :            : /// // Equivalent of MPI_Comm_barrier
     177                 :            : /// comm->barrier ();
     178                 :            : /// \endcode
     179                 :            : ///
     180                 :            : /// Comm's communication methods that actually send or receive data
     181                 :            : /// accept that data as an array of \c char.  You should never call
     182                 :            : /// these methods directly.  Instead, you should use the nonmember
     183                 :            : /// "helper" functions in Teuchos_CommHelpers.hpp.  These methods are
     184                 :            : /// templated on the \c Packet type, that is, the type of data you
     185                 :            : /// want to send or receive.  See the example below.
     186                 :            : ///
     187                 :            : /// \section Teuchos_Comm_Handle Treat <tt>RCP<const Comm<int> ></tt> like an opaque handle
     188                 :            : ///
     189                 :            : /// You should consider an <tt>RCP<const Comm<int> ></tt> as
     190                 :            : /// equivalent to the MPI_Comm opaque handle, except that the RCP also
     191                 :            : /// does reference counting to ensure memory safety when using the
     192                 :            : /// same communicator in different parts of the code.  That is,
     193                 :            : /// copying the RCP does not create a new communicator; the following
     194                 :            : /// two codes do about the same thing, except with a different syntax
     195                 :            : /// (and reference counting in the second code).
     196                 :            : ///
     197                 :            : /// Raw MPI_Comm handles:
     198                 :            : /// \code
     199                 :            : /// MPI_Comm comm = ...;
     200                 :            : /// // sameComm is THE SAME COMMUNICATOR as comm.
     201                 :            : /// MPI_Comm sameComm = comm;
     202                 :            : /// \endcode
     203                 :            : ///
     204                 :            : /// Reference-counted pointers to Comm:
     205                 :            : /// \code
     206                 :            : /// RCP<const Comm<int> > comm = ...;
     207                 :            : /// // *sameComm is THE SAME COMMUNICATOR as *comm.
     208                 :            : /// RCP<const Comm<int> > sameComm = comm;
     209                 :            : /// \endcode
     210                 :            : ///
     211                 :            : /// If you want to make a "new communicator" rather than just "copy
     212                 :            : /// the handle," you should call the duplicate() method.  This has the
     213                 :            : /// same behavior as MPI_Comm_dup (which see).
     214                 :            : ///
     215                 :            : /// The "reference counting" feature means that the subclass of Comm
     216                 :            : /// will take care of freeing the underlying MPI_Comm (and any other
     217                 :            : /// data structures it may use) by calling MPI_Comm_free if necessary,
     218                 :            : /// once the reference count of the RCP goes to zero.
     219                 :            : ///
     220                 :            : /// \warning Do <i>not</i> pass around subclasses of Comm by value!
     221                 :            : ///   Comm or its subclasses by themselves do not have handle
     222                 :            : ///   semantics.  Their copy constructors likely do not behave as you
     223                 :            : ///   would expect if the classes had handle semantics.
     224                 :            : ///
     225                 :            : /// \section Teuchos_Comm_How How do I make a Comm?
     226                 :            : ///
     227                 :            : /// Comm works whether or not you have build Trilinos with MPI
     228                 :            : /// support.  If you want to make a "default" Comm that is the
     229                 :            : /// equivalent of MPI_COMM_WORLD, but you don't know if your Trilinos
     230                 :            : /// with MPI enabled, you may use GlobalMPISession to call MPI_Init if
     231                 :            : /// necessary, and DefaultComm to "get a default communicator."  For
     232                 :            : /// example:
     233                 :            : /// int main (int argc, char* argv[]) {
     234                 :            : ///   using Teuchos::Comm;
     235                 :            : ///   using Teuchos::DefaultComm;
     236                 :            : ///   using Teuchos::RCP;
     237                 :            : ///
     238                 :            : ///   // This replaces the call to MPI_Init.  If you didn't
     239                 :            : ///   // build with MPI, this doesn't call MPI functions.
     240                 :            : ///   Teuchos::GlobalMPISesssion session (&argc, &argv, NULL);
     241                 :            : ///   // comm is the equivalent of MPI_COMM_WORLD.
     242                 :            : ///   RCP<const Comm<int> > comm = DefaultComm<int>::getComm ();
     243                 :            : ///
     244                 :            : ///   // ... use comm in your code as you would use MPI_COMM_WORLD ...
     245                 :            : ///
     246                 :            : ///   // We don't need to call MPI_Finalize, since the
     247                 :            : ///   // destructor of GlobalMPISession does that for us.
     248                 :            : ///   return EXIT_SUCCESS;
     249                 :            : /// }
     250                 :            : /// \endcode
     251                 :            : /// This code works whether or not you built Trilinos with MPI
     252                 :            : /// support.  It is not necessary to use GlobalMPISession, but it's
     253                 :            : /// useful so you don't have to remember to call MPI_Finalize.  If you
     254                 :            : /// don't want to use GlobalMPISession, you can still call
     255                 :            : /// <tt>DefaultComm<int>::getComm()</tt>, though you must have called
     256                 :            : /// MPI_Init first if you build Trilinos with MPI support.
     257                 :            : /// Furthermore, if you know MPI is present, you don't need to use
     258                 :            : /// DefaultComm.  You may simply pass MPI_COMM_WORLD directly to
     259                 :            : /// MpiComm, like this:
     260                 :            : /// \code
     261                 :            : /// RCP<const Comm<int> > comm = rcp (new MpiComm (MPI_COMM_WORLD));
     262                 :            : /// \endcode
     263                 :            : /// You may also pass an arbitrary MPI_Comm directly into MpiComm's
     264                 :            : /// constructor, though you are responsible for freeing it after use
     265                 :            : /// (via MPI_Comm_free) if necessary.  You may automate the freeing
     266                 :            : /// of your MPI_Comm by using OpaqueWrapper (which see).
