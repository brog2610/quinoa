Const ptr to physical variable access dispatch. Public interface to physical variable access. cptr() and cvar() are intended to be used together in case component and offset would be expensive to compute for data access via the function call operator. In essence, cptr() returns part of the address known based on component and offset and intended to be used in a setup phase. Then cvar() takes this partial address and finishes the address calculation given the particle id. The following two data accesses are equivalent (modulo constness): real& value = operator()( par, comp, offs ); const real* p = cptr( comp, offs ); const real& value = cvar( p, par );

Parameters:
[in]componentComponent index, i.e., position of a scalar within a system
[in]offsetSystem offset specifying the position of the system of equations among other systems
Returns:
Pointer to particle data of type tk::real for use with cvar()
See also:
Check out Statistics::setupOrdinary() and Statistics::accumulateOrd() in Statistics/Statistics.C to see cptr() and cvar() in action.

J. Bakosi

Data access dispatch. Public interface to particle data access. Use it as ParProps( p, c, o ), where p is the particle index, c is component index specifying the scalar equation within a system of equations, and o is the offset specifying the position at which the system resides among other systems.
