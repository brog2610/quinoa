00146 //! Pack/Unpack test runner. This Pack/Unpack method (re-)creates the
00147 //! test runner singleton on all processing elements. Therefore we circumvent
00148 //! Charm's usual pack/unpack for this type, and thus sizing does not make
00149 //! sense: sizing is a no-op. We could initialize the stack in UnitTestDriver's
00150 //! constructor and let this function re-create the runner only when unpacking,
