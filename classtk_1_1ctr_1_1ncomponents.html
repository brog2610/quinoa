Detailed Description

template<typename... Tags>
class tk::ctr::ncomponents< Tags >

Number of components storage. All this trickery with boost::mpl allows the code below to be generic. As a result, adding a new component requires adding a single line (a tag and its type) to the already existing list, see typedefs 'ncomps'. The member functions, doing initialization, computing the number of total components, the offset for a given tag, and computing the offset map, need no change -- even if the order of the number of components change.
