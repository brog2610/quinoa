00044 //! \brief Generic definition of a keyword
00045 //! \details A keyword is a struct that collects the information that makes up a
00046 //!    keyword. The requirement on the first template argument, Info, is that it
00047 //!    must define the name(), shortDescription(,) and longDescription() member
00048 //!    functions returning compile-time (static) std::strings. The
00049 //!    shortDescription() member function is used to return a short description
00050 //!    of what the keyword is used for, while the longDescription() member
00051 //!    function is used for a longer, e.g., a paragraph-long, description on
00052 //!    what the keyword can be used for and how it can and should be used. The
00053 //!    last template parameters are a list of character constants, specifying
00054 //!    the case-sensitive characters that make up the keyword, which is then
00055 //!    matched by the parser. The keyword must be at least one character long,
00056 //!    but otherwise its length is only limited by the compiler's recursion
00057 //!    handling capability of variadic templates. While the name(),
00058 //!    shortDescription() and longDescription() member functions of Info are
00059 //!    required, there are also optional ones, such as
00060 //!    Info::exptect::description(), which, if defined, must also be static and
00061 //!    return a std::string, describing the type the particular keyword expects
00062 //!    during parsing. This is optional since not every keyword expects a value
00063 //!    (or values) of a particular type. For example, the keyword 'end' is
00064 //!    simply used to close a block, and what follows does not have a
00065 //!    relationship to the keyword. A counterexample is is 'title', which
00066 //!    expects a double-quoted string.
00067 //! \see For example client-code and more detailed documentation on the possible
00068 //!    fields, see Control/Keywords.h.
00069 //! \author J. Bakosi
