00563     /** @name Host signal calls
00564       * \brief These functions signal back to the host via a global reduction
00565       *   originating from each PE branch
00566       * \details Singal calls contribute to a reduction on all branches (PEs)
00567       *   of LinSysMerger to the host, e.g., inciter::CProxy_Conductor, given by
00568       *   the template argument HostProxy. The signal functions are overloads on
00569       *   the specialization, e.g., inciter::CProxy_Conductor, of the
00570       *   LinSysMerger template. They create Charm++ reduction targets via
00571       *   creating a callback that invokes the typed reduction client, where
00572       *   host is the proxy on which the reduction target method, given by the
00573       *   string followed by "redn_wrapper_", e.g., init(), is called upon
00574       *   completion of the reduction.
00575       *
00576       *   Note that we do not use Charm++'s CkReductionTarget macro here,
00577       *   but instead explicitly generate the code that that macro would
00578       *   generate. To explain why, here is Charm++'s CkReductionTarget macro's
00579       *   definition, given in ckreduction.h:
00580       *   \code{.cpp}
00581       *      #define CkReductionTarget(me, method) \
00582       *        CkIndex_##me::redn_wrapper_##method(NULL)
00583       *   \endcode
00584       *   This macro takes arguments 'me' (a class name) and 'method' a member
00585       *   function of class 'me' and generates the call
00586       *   'CkIndex_<class>::redn_wrapper_<method>(NULL)'. With the overloads the
00587       *   signal2* functions generate, we do the above macro's job for
00588       *   LinSysMerger specialized by HostProxy, hard-coded here, as well its
00589       *   reduction target. This is required since
00590       *    * Charm++'s CkReductionTarget macro's preprocessing happens earlier
00591       *      than type resolution and the string of the template argument would
00592       *      be substituted instead of the type specialized (which is not what
00593       *      we want here), and
00594       *    * the template argument class, e.g, CProxy_Conductor, is in a
00595       *      namespace different than that of LinSysMerger. When a new class is
00596       *      used to specialize LinSysMerger, the compiler will alert that a new
00597       *      overload needs to be defined.
00598       *
00599       * \note This simplifies client-code, e.g., inciter::Conductor, which now
00600       *   requires no explicit book-keeping with counters, etc. Also a reduction
00601       *   (instead of a direct call to the host) better utilizes the
00602       *   communication network as computational nodes can send their aggregated
00603       *   contribution to other nodes on a network instead of all chares sending
00604       *   their (smaller) contributions to the same host, (hopefully)
00605       *   implemented using a tree among the PEs.
00606       * \see http://charm.cs.illinois.edu/manuals/html/charm++/manual.html,
00607       *   Sections "Processor-Aware Chare Collections" and "Chare Arrays".
00608       * */
00609     ///@{
00610     //! \brief Signal back to host that the initialization of the linear system
00611     //!   is complete

00613       using inciter::CkIndex_Conductor;
00614       Group::contribute(
