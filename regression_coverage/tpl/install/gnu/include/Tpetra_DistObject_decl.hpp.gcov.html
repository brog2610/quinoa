      74                 :            :   /// \brief Base class for distributed Tpetra objects that support
      75                 :            :   ///   data redistribution.
      76                 :            :   ///
      77                 :            :   /// DistObject is a base class for all Tpetra distributed global
      78                 :            :   /// objects, including CrsMatrix and MultiVector.  It provides the
      79                 :            :   /// basic mechanisms and interface specifications for importing and
      80                 :            :   /// exporting operations using Import and Export objects.
      81                 :            :   ///
      82                 :            :   /// \tparam LocalOrdinal The type of local IDs.  Same as Map's
      83                 :            :   ///   \c LocalOrdinal template parameter.  This should be an integer
      84                 :            :   ///   type, preferably signed.
      85                 :            :   ///
      86                 :            :   /// \tparam GlobalOrdinal The type of global IDs.  Same as Map's
      87                 :            :   ///   \c GlobalOrdinal template parameter.  Defaults to the same type
      88                 :            :   ///   as \c LocalOrdinal.  This should also be an integer type,
      89                 :            :   ///   preferably signed.
      90                 :            :   ///
      91                 :            :   /// \tparam Node Same as Map's \c Node template parameter.  Defaults
      92                 :            :   ///   to the default Kokkos Node type.
      93                 :            :   ///
      94                 :            :   /// \tparam classic DO NOT SET THIS EXPLICITLY.  This template
      95                 :            :   ///   parameter only exists for backwards compatibility.  It must
      96                 :            :   ///   always be false.
      97                 :            :   ///
      98                 :            :   /// \section Tpetra_DistObject_Summary Summary
      99                 :            :   ///
     100                 :            :   /// Most Tpetra users will only use this class' methods to perform
     101                 :            :   /// data redistribution for subclasses such as CrsMatrix,
     102                 :            :   /// MultiVector, and Vector.  DistObject provides four methods for
     103                 :            :   /// redistributing data: two versions of <tt>doImport()</tt>, and
     104                 :            :   /// two versions of <tt>doExport()</tt>.  Import operations
     105                 :            :   /// redistribute data from a nonoverlapping (one-to-one)
     106                 :            :   /// distribution to a possibly overlapping distribution.  Export
     107                 :            :   /// operations redistribute data from a possibly overlapping
     108                 :            :   /// distribution to a nonoverlapping (one-to-one) distribution.
     109                 :            :   /// Once you have precomputed a data redistribution plan (an Import
     110                 :            :   /// or Export object), you may use the plan to redistribute an input
     111                 :            :   /// object's data into this object, by calling one of these methods.
     112                 :            :   /// The input object of <tt>doImport()</tt> or <tt>doExport()</tt>
     113                 :            :   /// is always the "source" of the redistribution operation, which
     114                 :            :   /// sends the data.  The <tt>*this</tt> object is the target, which
     115                 :            :   /// receives and combines the data.  It has the distribution given
     116                 :            :   /// by <tt>this->getMap()</tt>.
     117                 :            :   ///
     118                 :            :   /// \section Tpetra_DistObject_FwdRev Forward or reverse redistribution modes
     119                 :            :   ///
     120                 :            :   /// Both Import and Export operations occur in two modes: forward
     121                 :            :   /// and reverse.  Forward mode is the usual case, where you are
     122                 :            :   /// calling a method with its matching plan type
     123                 :            :   /// (<tt>doImport()</tt> for an Import plan, or <tt>doExport()</tt>
     124                 :            :   /// for an Export plan).  In that case, the input DistObject must
     125                 :            :   /// have the same Map as the source Map of the plan, and the target
     126                 :            :   /// DistObject must have the same Map as the target Map of the plan.
     127                 :            :   /// Reverse mode is also possible, where you call a method with the
     128                 :            :   /// opposite plan type (<tt>doImport()</tt> for an Export plan, or
     129                 :            :   /// <tt>doExport()</tt> for an Import plan).  In that case, the
     130                 :            :   /// source DistObject's Map must be the same as the target Map of
     131                 :            :   /// the plan, and the target DistObject's Map must be the same as
     132                 :            :   /// the source Map of the plan.  If you call <tt>doImport()</tt>, we
     133                 :            :   /// still call this an Import operation, even if you are using an
     134                 :            :   /// Export plan in reverse.  Similarly, if you call
     135                 :            :   /// <tt>doExport()</tt>, we call this an Export operation.
     136                 :            :   ///
     137                 :            :   /// Most users will want to use forward mode.  However, reverse mode
     138                 :            :   /// is useful for some applications.  For example, suppose you are
     139                 :            :   /// solving a nonlinear partial differential equation using the
     140                 :            :   /// finite element method, with Newton's method for the nonlinear
     141                 :            :   /// equation.  When assembling into a vector, it is convenient and
     142                 :            :   /// efficient to do local assembly first into a vector with an
     143                 :            :   /// overlapping distribution, then do global assembly via forward
     144                 :            :   /// mode Export into a vector with a nonoverlapping distribution.
     145                 :            :   /// After the linear solve, you may want to bring the resulting
     146                 :            :   /// nonoverlapping distribution vector back to the overlapping
     147                 :            :   /// distribution for another update phase.  This would be a reverse
     148                 :            :   /// mode Import, using the precomputed Export object.
     149                 :            :   ///
     150                 :            :   /// Another use case for reverse mode is in CrsMatrix, for the
     151                 :            :   /// transpose version of distributed sparse matrix-vector multiply
     152                 :            :   /// ("mat-vec").  Non-transpose mat-vec (a function from the domain
     153                 :            :   /// Map to the range Map) does an Import to bring in the source
     154                 :            :   /// vector's data from the domain Map to the column Map of the
     155                 :            :   /// sparse matrix, and an Export (if necessary) to bring the results
     156                 :            :   /// from the row Map of the sparse matrix to the range Map.
     157                 :            :   /// Transpose mat-vec (a function from the range Map to the domain
     158                 :            :   /// Map) uses these precomputed Import and Export objects in reverse
     159                 :            :   /// mode: first the Export in reverse mode to Import the source
     160                 :            :   /// vector's data to the row Map, and then the Import in reverse
     161                 :            :   /// mode to Export the results to the domain Map.  Reverse mode lets
     162                 :            :   /// us reuse the precomputed data redistribution plans for the
     163                 :            :   /// transpose case.
     164                 :            :   ///
     165                 :            :   /// \section Tpetra_DistObject_ImplSubclass How to implement a subclass
